squash + rebase

  • Минусы:
    1. Сложнее понять работу rebase
      • Разработчики используют gitflow, мейнтейнер - squash/rebase
    2. Изменение истории - потенциально опасная операция
      • git push --force-with-lease не затрет чужие коммиты
    3. История develop'а может измениться
      • git pull --rebase
    4. Resolve одних и тех же конфликтов
      • git rerere должен помочь
